Case Study: Module Concepts

Students prove their thinking process and connect a technical concept to their own device.

Traditional Assessment

Fill in the Blank

Students answer a recall-style question such as:

My IPv4 address is: ________________

This checks whether the student can copy a value, but it does not verify understanding or connect the concept to their own computing environment.

Resilient Update

Define, Locate, Document, Reflect

Students define an IP address in their own words, explain how it affects their device on a network, and include a screenshot showing their local IP address.

  • Define IP address in your own words.
  • Explain how the address helps the device communicate on a local network.
  • Show a screenshot of the student's current local IP address with any sensitive details hidden.
  • Submit a short process note describing where they found the address and what they checked before submitting.

Reflection Questions

What does your IP address allow your device to do?

Connect the address to communication, routing, and identifying the device on the local network.

What could AI explain generally, but not prove for you?

AI can define IP addressing, but it cannot prove what address appeared on the student's own device at the time of the assignment.

What did the screenshot verify?

It verifies that the student performed the local lookup and can connect the course concept to their own network environment.

Assignment Prompt:
Define what an IP address is in your own words. Explain how your local IP address affects your device when it communicates on a network. Then locate your current local IP address, take a screenshot, and add a short process note explaining where you found it and what the screenshot proves.

Reflection Questions:
- What does your IP address allow your device to do?
- What could AI explain generally, but not prove for you?
- What did the screenshot verify?

Case Study: Python Mini Project

Students may use AI coding help, but they must prove they understand the program they submit.

Traditional Assessment

Submit a Working Script

Students submit a Python file that converts a temperature, calculates a total, or prints formatted output.

Write a Python script that converts Fahrenheit to Celsius.

Modern AI tools can generate a correct small script instantly. A working file alone does not show whether the student can read, modify, or explain the code.

Resilient Update

Code Submission + 3-Minute Explanation

Students submit a small Python program and an unedited 3-minute video explaining how it works.

  • Build a script that asks for a network speed in Mbps and estimates download time for a file size.
  • Include a screenshot showing the program running with two different inputs.
  • In a 3-minute video, explain one variable, one calculation, and one change they would make if the file size were entered in gigabytes instead of megabytes.
  • AI assistance is allowed, but students must be able to defend the code.

Reflection Questions

Which line of code is most important to the calculation?

Students should identify the formula line and explain how input values affect the result.

What would change if the user entered gigabytes?

The student should describe the unit conversion and where it belongs in the program.

What can the video reveal that the code file cannot?

It reveals whether the student can explain, troubleshoot, and modify the submitted work.

Assignment Prompt:
Create a Python script that asks the user for a file size in megabytes and a network speed in Mbps, then estimates the download time. Submit your .py file, a screenshot showing two successful test runs, and an unedited 3-minute video explaining one variable, one calculation, and how you would modify the script if file size were entered in gigabytes.

Reflection Questions:
- Which line of code is most important to the calculation?
- What would change if the user entered gigabytes?
- What can the video reveal that the code file cannot?

Case Study: Network Explanation Audit

Students act as reviewers who must catch and correct flawed AI-generated technical explanations.

Traditional Assessment

Definitions Paragraph

Students write a paragraph defining public IP address, private IP address, subnet mask, default gateway, and DNS server.

This is easy for AI to generate and hard to authenticate because the final paragraph may look polished without student understanding.

Resilient Update

Audit a Flawed AI Explanation

Students receive a short AI-generated explanation containing planted errors. They must identify, correct, and justify each correction using course materials.

"A assigned by your ISP. A subnet mask , while the default gateway ."

Reflection Questions

Which error looked most believable?

Students should explain why a wrong claim sounded plausible and how they verified it.

What source confirmed your correction?

Students should cite course notes, textbook pages, or lab instructions instead of relying on another AI answer.

How does auditing differ from writing from scratch?

Auditing makes the student responsible for judgment, verification, and correction.

Assignment Prompt:
Review the provided AI-generated explanation of IP addressing, subnet masks, default gateways, and DNS. Mark at least three incorrect or misleading claims, explain why each claim is wrong using course materials, and rewrite the passage correctly in your own words.

Reflection Questions:
- Which error looked most believable?
- What source confirmed your correction?
- How does auditing differ from writing from scratch?

Case Study: Wi-Fi Troubleshooting

Students document a real connection context that AI cannot know without their local evidence.

Traditional Assessment

Common Wi-Fi Problems Worksheet

Students list possible reasons Wi-Fi might not work: weak signal, wrong password, router problem, no internet service, or DNS issue.

The answers are useful, but generic. AI can produce a complete troubleshooting list without any student touching a device.

Resilient Update

Local Wi-Fi Evidence Walkthrough

Students document a real or simulated troubleshooting process on a device they use.

  • Show the Wi-Fi connection status with private information hidden when needed.
  • Show the local IP address assigned while connected to Wi-Fi.
  • Run one connectivity test: default gateway ping, browser test, or OS network diagnostic.
  • Explain what they would check next if Wi-Fi showed connected but webpages did not load.

Apply the Method

Check each local constraint that makes the assignment hard to fake with generic AI text.

Resilience Features Present:
0%

Reflection Questions

What proved that the device was connected to Wi-Fi?

Students should point to connection status, SSID context, IP assignment, or operating system network details.

What proved whether internet access was working?

Students should connect the evidence to a browser test, ping result, diagnostic result, or gateway behavior.

What could AI describe, but not verify?

AI can describe Wi-Fi troubleshooting, but it cannot verify the student's exact device, network state, and test results.

Assignment Prompt:
Document a Wi-Fi troubleshooting check on a device you use. Include evidence of the Wi-Fi connection, your local IP address, and one connectivity test. Then explain what each piece of evidence shows and what you would check next if Wi-Fi appeared connected but webpages did not load.

Reflection Questions:
- What proved that the device was connected to Wi-Fi?
- What proved whether internet access was working?
- What could AI describe, but not verify?
